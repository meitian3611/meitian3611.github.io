每日3道面试题

一，HTML，HTTP，web综合相关问题

1.如何进行性能优化

在html方面，我们可以减少http请求，图片进行压缩，使用css精灵图，采用语义化标签

引入的css文件统一放在页面顶部，js文件统一放在页面底部 js中少使用全局变量

2.语义化的理解

用正确的标签做正确的事情

语义化可以使得页面内容更加的结构化，便于对浏览器，搜索引擎解析 优化seo

使源码更易维护，便于理解

3.HTTP的几种请求方式

  • GET方法

发送一个请求来取得服务器上的某一资源

  • POST方法

向url指定的资源提交数据或附加新的数据

  • PUT
  • HEAD 只会请求页面首部
  • DELETE 删除服务器上面的某个资源
  • OPTIONS
  • CONNECT

4.对浏览器内核的理解

浏览器内核主要分为两大部分：渲染引擎和 JS 引擎

渲染引擎主要负责取得网页内容，加载css，计算网页显示方式，不同浏览器的内核不同，渲染出的页面效果也不相同

JS引擎是解析和执行JavaScript来实现网页的动态效果

内核普遍指向渲染引擎

5.cookie，seesionStorage 和 localStorage的区别

  • cookie是储存在本地终端上的数据

    cookie会始终携带在同源的http请求中，在浏览器和服务器之间传递

  • seesionStorage 和 localStorage不会发送数据给服务器，仅在本地保存

  • 存储大小：cookie数据大小不能超过4K，本地存储可以达到5M

  • 时间：localStorage数据永久存在，除非主动删除

    sessionStorage 浏览器窗口关闭后自动删除

  cookie 可以设置有效时间

6.Doctype作用? 严格模式与混杂模式如何区分？它们有何意义?

  • <!DOCTYPE> 声明位于文档中的最前面，处于<HTML> 标签之前。告知浏览器的解析器， 用什么文档类型 规范来解析这个文档
  • 严格模式的排版和 JS 运作模式是 以该浏览器支持的最高标准运行
  • 在混杂模式中，页面以宽松的向低版本兼容的方式显示。模拟老式浏览器的行为以防止站点无法工作。 DOCTYPE不存在或格式不正确会导致文档以混杂模式呈现

7.前端需要注意哪些SEO

  • 语义化HTML代码，使用W3C规范，使得搜索引擎更容易解析网页
  • 重要的HTML代码放在最前面，以便于搜索引擎从上到下抓取
  • 少用iframe，搜索引擎不会抓取iframe的内容
  • 非装饰图片都要加 alt

8.web标准和W3C标准是什么

  • 标签闭合，标签小写，不乱嵌套，使用外链css和js，结构行为样式分离

9.iframe有什么缺点

  • iframe会阻塞主页面的onload事件
  • 搜索引擎无法解析，不利于SEO
  • iframe和主页面共享连接池，但浏览器对相同域会有限制，会影响页面的并行加载
  • 可以通过javascript动态给iframe添加src属性，解决问题
